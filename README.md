## ✅ 트위버: 여행지 추천 기능이 있는 커뮤니티 사이트
 트위버는 여행지 추천 및 여행 커뮤니티 플랫폼 서비스입니다. 의미있는 프로젝트가 되기 위해 4명의 팀원들과 최대한 도움이 되는 코드 리뷰를 하기 위해 노력했고, 스터디를 진행했습니다. 기능 개발에 많은 고민을 했고, 오류 해결을 위해 모든 팀원과 함께 밤을 새기도 하며 진심을 다해 프로젝트에 임했습니다. 총 인원은 5명(F.E. 2 + B.E. 3)으로, 저는 백엔드 개발을 맡았습니다.

[README, WIKI](https://github.com/ValueWith/ValueWith_BE)를 통해 상세정보를 확인할 수 있습니다.

사이트 기능, 전체 개요는 [같이의가치팀 README](https://github.com/ValueWith)와 [본 레포지토리의 포트폴리오 pdf 파일](https://github.com/eod940/portfolio-eod940/blob/main/%EB%B0%B1%EC%97%94%EB%93%9C_%EA%B0%9C%EB%B0%9C%EC%9E%90_%EC%9D%B4%EB%8C%80%EC%98%81_%ED%8F%AC%ED%8A%B8%ED%8F%B4%EB%A6%AC%EC%98%A4.pdf)을 통해 확인하실 수 있습니다.
> 기간: 2023.10 ~ 2023.04 (5개월)

**[ 주요 업무 ]**

- Spring boot, JPA, MySQL 기반 REST API 서버 개발
- ERD, UseCase, 주요 핵심 기능 설계 (Github Wiki 작성)
- 회원, 후기, 채팅 도메인 개발

**[ 개발 내용 ]**

- JWT 토큰 발급 및 Access Token, Refresh Token 기반 로그인 기능 개발
- ControllerAdvice의 예외 처리 범위 문제를 Filter에서 해결해가는 과정 경험
- 회원가입에 필요한 인증 프로세스를 위해 Spring Cloud의 OpenFeign 학습 및 적용
- 실시간 채팅을 위한 Web Socket, STOMP 적용
- JPA 사용중 맞닥뜨린 N+1 이슈, OSIV 관련 생명주기 이슈의 해결 과정 경험
- 배포 및 CI/CD 파이프라인 구축
- 서버 배포시 EC2 인스턴스 메모리 부족 문제를 Swap 메모리를 통해 해결
- S3 버킷에 적재된 빌드 파일 용량 최적화 고민을 통해 저장소 총 용량 61% 절감 개선

**[ 사용 기술 ]**

- Java, Spring Boot, Spring Security, Spring Cloud OpenFeign, MySQL, Redis, OAuth2
- AWS EC2, Codedeploy, S3, Cloud front,  Route53, Elastic load balancing
- Github, Slack, Discord, Notion
